# Thinkers Roundtable Overview

Thinkers Roundtable is a prompt system for hard decisions.

Instead of asking an AI for a smooth answer, you ask it to host a structured disagreement between different decision lenses. Each lens attacks the problem from a different angle. The goal is not consensus. The goal is to expose what you are missing.

## The Default Four

These four lenses are present by default:

| Lens | Role in the room |
|---|---|
| Elon Musk | Breaks the problem down to physical limits, cost structure, and unnecessary requirements. |
| Charlie Munger | Looks for stupidity, incentives, fatal flaws, and ways the decision can fail. |
| Nassim Taleb | Focuses on ruin risk, asymmetry, fragility, skin in the game, and survival. |
| Naval Ravikant | Tests leverage, specific knowledge, desire, freedom, and long-term compounding. |

Their tension is the engine:

- Musk pushes speed and first principles.
- Munger blocks obvious stupidity.
- Taleb protects against irreversible downside.
- Naval asks whether the game is worth playing at all.

## The Optional Seven

Bring these in when the decision calls for them:

| Lens | Use when |
|---|---|
| Richard Feynman | You may not truly understand the thing you are deciding about. |
| Steve Jobs | You need focus, product taste, and end-to-end experience judgment. |
| Andrej Karpathy | The decision involves AI, technical reliability, or demo-vs-production risk. |
| Paul Graham | You need to write the idea clearly, test it early, or judge startup direction. |
| Ilya Sutskever | You need to distinguish real understanding from memorization or incremental improvement from phase change. |
| Donald Trump | The decision is a negotiation, public narrative, or power game. |
| MrBeast | The decision depends on attention, retention, packaging, and whether people will care. |

## The Six Stages

1. Problem diagnosis.
2. Roundtable debate.
3. Feasibility check.
4. Conflict arbitration.
5. Bias control.
6. Integrated output.

The output should end with one concrete step you can take within a week.

## What This Is Not

This is not a replacement for professional advice.

This is not an impersonation tool.

This is not a way to outsource responsibility.

It is a thinking aid for decisions where one friendly answer is not enough.
